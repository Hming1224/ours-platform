推薦版型

純長捲動 + 頂部錨點

↑②事實 → ③共識放最上：最無爭議、跨立場都同意的先講，建立共同基礎。
↓④光譜 → ⑤意見往後：進入分歧，但已有共識墊底。
＋⑥來源 → ⑦時間序給想深究的人鑽取與脈絡；⑧⑨延伸到討論頁與相關議題。
生成條件：②事實一定有；⑤意見要有使用者討論才生成 → 需畫「尚無 opinion」空狀態（見下）。
⑤ 意見摘要 opinion — 兩版型 + 空狀態

落實 IA 規則：觀點最多 3 種，每種可展開看來源。橫向卡片為推薦。

規則對齊

對照 IA DISCUSSION 規則與簡報定調。

✓ 已對齊（這次設計已遵守）

  • 有聲量才進議題頁 → 卡片、標題區都以聲量為核心訊號
  • 新聞只摘事實 → ②facts 標明「只摘事實」
  • 新聞＋使用者才摘 opinion → ⑤有生成條件 + 空狀態
  • 觀點最多 3 種、可看來源 → ⑤兩版型都 3 種、可展開
  • 先共識後分歧 → 捲動順序事實/共識在前、光譜/意見在後
  • 觀點光譜走輕量 → 分布條，不塞複雜大圖表
  • AI 是助手不是裁判 → 光譜文案強調「整體分布、非評判單一媒體」
  • 不混進首頁瀑布流 → 議題為獨立 Tab 場域

○ 待團隊討論（設計只預留位置）

  • 熱度演算法（誰進議題頁、列表排序）→ 先用佔位排序
  • 「最新議題」定義 → 列表頁留「最新」過渡區佔位
  • 新聞品質評估的呈現 → ⑥來源先預留「品質標記」位
  • 「有幫助」如何餵橋接式排序 → 互動已埋點，演算法待定
